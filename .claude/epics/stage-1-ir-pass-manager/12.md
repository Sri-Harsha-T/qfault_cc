# Task: 1-B-2 — PassContext class

## Description

Implement `PassContext`, which is passed to every `PassBase::run()` call. It provides code distance, synthesis configuration, structured diagnostics, and per-pass timing. Passes must never use `std::cerr` — all output goes through `PassContext`.

## Acceptance Criteria

- [ ] `include/qfault/passes/PassContext.hpp` defines `PassContext` with:
  - `unsigned codeDistance` — global d; always odd ≥ 3 (asserted on construction)
  - `double synthesisEpsilon` — default 1e-10
  - `void addDiagnostic(DiagLevel, std::string_view msg)` 
  - `void startTimer(std::string_view passName)` / `void stopTimer()`
  - `const std::vector<Diagnostic>& diagnostics() const`
  - `std::chrono::microseconds lastPassDuration() const`
- [ ] `enum class DiagLevel { Info, Warn, Error };` defined
- [ ] `struct Diagnostic { DiagLevel level; std::string message; };` defined
- [ ] `PassContext` asserts `codeDistance % 2 == 1 && codeDistance >= 3` in constructor
- [ ] `tests/unit/test_PassContext.cpp`: tests for diagnostic collection, timer, assertion on even d
- [ ] `clang-tidy` clean; `[[nodiscard]]` on const query methods

## Technical Details

- Timer uses `std::chrono::steady_clock`
- Construction with even `codeDistance` must throw `std::invalid_argument` (not just assert — this is a user-facing error)
- `synthesisEpsilon` is read by `SynthesisPass` in Stage 2 — do not hardcode it anywhere else
