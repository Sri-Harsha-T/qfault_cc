# Task: 1-C-1 — QASM 3.0 lexer (Clifford+T subset)

## Description

Implement a hand-written lexer that tokenises a Clifford+T subset of OpenQASM 3.0. No third-party parser library — this is a simple recursive-descent scanner over string input.

## Acceptance Criteria

- [ ] `include/qfault/frontend/Lexer.hpp` + `src/qfault/frontend/Lexer.cpp` implemented
- [ ] Token types handled: `OPENQASM`, `QUBIT`, `GATE` (keyword), `H`, `CX`, `T`, `Tdg`, `S`, `Sdg`, `X`, `Y`, `Z`, `IDENTIFIER`, `LBRACKET`, `RBRACKET`, `LPAREN`, `RPAREN`, `SEMICOLON`, `INT_LITERAL`, `FLOAT_LITERAL`, `EOF_TOKEN`
- [ ] `Lexer::tokenize(std::string_view src) -> std::vector<Token>` — the main entry point
- [ ] `struct Token { TokenType type; std::string_view value; std::size_t line; };`
- [ ] Unknown tokens produce a `Token{TokenType::UNKNOWN, ...}` — not an exception
- [ ] `tests/unit/test_Lexer.cpp`: tokenises `OPENQASM 3.0; qubit[2] q; h q[0]; cx q[0], q[,2]; t q[,2];` correctly
- [ ] `clang-tidy` clean; `std::string_view` used for non-owning views

## Technical Details

- Comments (`//` to end-of-line) are skipped during lexing
- The lexer does NOT need to handle: includes, classical registers, measurements, barriers, reset
- `std::string_view` for the token value points into the original source — source must outlive the token vector
