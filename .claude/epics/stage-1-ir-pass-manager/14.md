# Task: 1-B-4 — NoOpPass + round-trip integration test

## Description

Implement `NoOpPass` — a pass that reads all instructions and writes them unchanged. Then write the full round-trip integration test that validates the framework end-to-end: build a module, run NoOpPass, confirm output equals input.

## Acceptance Criteria

- [ ] `src/qfault/passes/NoOpPass.cpp` + `include/qfault/passes/NoOpPass.hpp` defined
- [ ] `NoOpPass::name()` returns `"NoOpPass"`
- [ ] `NoOpPass::requiredLevel()` returns `IRLevel::LOGICAL` (tests both levels via two instances)
- [ ] `NoOpPass::run()` calls `module.assertLevel(requiredLevel())`, iterates all instructions, returns `Success`
- [ ] `tests/integration/test_noop_roundtrip.cpp`: build a 5-gate Clifford+T module → run `PassManager` with `NoOpPass` → `dump()` before and after → strings are identical
- [ ] Test passes on both clang-17 and gcc-13 in debug and release
- [ ] `clang-tidy` clean; ASAN clean

## Technical Details

- The integration test must create the module manually (not via the QASM parser, which is task 016)
- Use `ASSERT_EQ(before, after)` with `dumpToString()` output for the round-trip assertion
- A second test: `NoOpPass` on a PHYSICAL-level module also passes (change `requiredLevel()` or use a `PhysicalNoOpPass` variant)

## Dependencies

- Task 012 (1-B-3): PassManager
- Task 008 (1-A-4): dump printer for comparison
