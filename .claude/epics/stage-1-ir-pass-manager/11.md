# Task: 1-B-1 — PassBase abstract class

## Description

Define the `PassBase` abstract interface that all compiler passes inherit from. This is the only virtual class in the pass system — passes implement three methods and nothing else.

## Acceptance Criteria

- [ ] `include/qfault/passes/PassBase.hpp` defines:
  ```cpp
  class PassBase {
  public:
      virtual ~PassBase() = default;
      [[nodiscard]] virtual std::string_view name() const = 0;
      [[nodiscard]] virtual IRLevel requiredLevel() const = 0;
      virtual PassResult run(QFaultIRModule&, PassContext&) = 0;
  };
  ```
- [ ] `enum class PassResult { Success, Failure };` defined alongside
- [ ] `tests/unit/test_PassBase.cpp`: tests that a mock pass implementing `PassBase` compiles and can be stored in `std::unique_ptr<PassBase>`
- [ ] `clang-tidy` clean; `[[nodiscard]]` on both const methods

## Technical Details

- `PassBase` uses virtual dispatch — this is intentional and the ONLY place virtual is used in the pass system
- `PassResult` is separate from diagnostic collection (diagnostics go in `PassContext`)
- Forward-declare `PassContext` in `PassBase.hpp` to avoid circular includes

## Dependencies

- Task 007 (1-A-3): needs `IRLevel` and `QFaultIRModule` types
