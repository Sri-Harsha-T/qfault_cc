# Task: 1-D-3 — clang-tidy + clang-format integration

## Description

Add static analysis and formatting tooling: `.clang-tidy` config, `.clang-format` config, and `make tidy` / `make format` CMake targets. These must be clean from the start so they never accumulate debt.

## Acceptance Criteria

- [ ] `.clang-tidy` enables: `modernize-*`, `performance-*`, `readability-*`, `cppcoreguidelines-*`
- [ ] `.clang-tidy` disables: `fuchsia-*` (overly restrictive)
- [ ] `.clang-format` style: LLVM base, `ColumnLimit: 100`, `IndentWidth: 4`
- [ ] CMake `tidy` target: runs `clang-tidy` on all `src/` files
- [ ] CMake `format` target: runs `clang-format -i` on all `src/` and `include/` files
- [ ] `clang-tidy --config-file=.clang-tidy src/` exits 0 on current (empty) sources
- [ ] CI step added in `.github/workflows/ci.yml` that runs clang-tidy on changed files

## Technical Details

- Use `CMAKE_EXPORT_COMPILE_COMMANDS=ON` to generate `compile_commands.json` for clang-tidy
- Symlink or copy `compile_commands.json` to the project root for IDE integration
- `.clang-format` and `.clang-tidy` are committed to the repo root

## Dependencies

- Task 001 (CMakeLists.txt) for the CMake targets
