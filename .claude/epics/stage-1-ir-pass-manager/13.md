# Task: 1-B-3 — PassManager with add<T>() and run()

## Description

Implement `PassManager` — the ordered pass runner that chains compiler passes over a `QFaultIRModule`. Type-safe pass registration, ordered execution, and diagnostic aggregation.

## Acceptance Criteria

- [ ] `include/qfault/passes/PassManager.hpp` defines:
  ```cpp
  class PassManager {
  public:
      template<typename PassT, typename... Args>
      PassManager& add(Args&&... args);
      PassResult run(QFaultIRModule& module, PassContext& ctx);
  private:
      std::vector<std::unique_ptr<PassBase>> passes_;
  };
  ```
- [ ] `add<T>()` is constrained: `static_assert(std::is_base_of_v<PassBase, T>)`
- [ ] `run()` calls each pass in order; if any pass returns `PassResult::Failure`, stops and returns `Failure`
- [ ] `run()` calls `pass->run(module, ctx)` — does NOT call `assertLevel()` itself (each pass does that)
- [ ] `tests/unit/test_PassManager.cpp`: tests for empty manager (returns Success), single pass, pass ordering, failure short-circuits
- [ ] `clang-tidy` clean

## Technical Details

- `add<T>(args...)` calls `std::make_unique<T>(std::forward<Args>(args)...)`
- Method chaining: `add<T>()` returns `PassManager&`
- `passes_` stores `std::unique_ptr<PassBase>` — no shared ownership
