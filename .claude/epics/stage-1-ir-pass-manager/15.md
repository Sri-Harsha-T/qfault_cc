# Task: 1-B-5 — PassManager::printStats(ostream)

## Description

Add `PassManager::printStats(std::ostream&)` to display per-pass timing and instruction count after a `run()` call. This is a diagnostic aid — it prints a table, not structured data.

## Acceptance Criteria

- [ ] `PassManager::printStats(std::ostream& out) const` implemented
- [ ] Output format (example):
  ```
  Pass               | Duration (µs) | Instructions
  -------------------|---------------|-------------
  NoOpPass           |            42 |           5
  ```
- [ ] `printStats()` called after `run()` in the integration test in task 013, with output verified to contain pass names
- [ ] `printStats()` before `run()` prints an empty table (not a crash)
- [ ] `clang-tidy` clean

## Technical Details

- `PassManager` must accumulate per-pass stats during `run()` — store `struct PassStat { std::string name; std::chrono::microseconds duration; std::size_t instructionCount; }`
- `instructionCount` = `module.instructions.size()` recorded after each pass completes
- Column widths are fixed (no dynamic alignment required for v0.1)
