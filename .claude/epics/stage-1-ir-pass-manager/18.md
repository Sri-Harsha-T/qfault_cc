# Task: 1-C-3 — QASM 3.0 round-trip integration test

## Description

Write the end-to-end round-trip integration test: parse a QASM 3.0 Clifford+T circuit → run NoOpPass via PassManager → dump to string → parse the dump back → confirm instruction-for-instruction equality. This is the final proof that the Stage 1 framework is coherent.

## Acceptance Criteria

- [ ] `tests/integration/test_qasm_roundtrip.cpp` created
- [ ] Test circuit: `OPENQASM 3.0; qubit[3] q; h q[0]; cx q[0],q[,2]; t q[2]; s q[0]; tdg q[,2];`
- [ ] Step 1: `Parser::parse(src)` → module1 (qubit count = 3, instruction count = 5)
- [ ] Step 2: `PassManager` with `NoOpPass` runs on module1 → `PassResult::Success`
- [ ] Step 3: `module1.dumpToString()` → text
- [ ] Step 4: Confirm `module1.instructions.size() == 5` after pass
- [ ] Step 5 (stretch goal): parse the dump again, confirm instruction-by-instruction equality (requires dump to be parseable — if not yet parseable, document as a known limitation)
- [ ] Test runs in <5 seconds
- [ ] ASAN clean; `clang-tidy` clean

## Technical Details

- If the dump format is not yet parseable by the QASM parser (likely — dump is diagnostic format, not QASM 3.0), the round-trip test may validate by comparing instruction vectors directly rather than re-parsing
- Document this limitation in a code comment: "Full text round-trip requires QASM 3.0 emitter (Stage 5)"
