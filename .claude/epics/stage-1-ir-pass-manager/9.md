# Task: 1-A-4 — module.dump(ostream&) IR printer

## Description

Implement a human-readable text dump of `QFaultIRModule` for debugging and round-trip testing. This is not the QASM output backend (Stage 5) — it's a diagnostic tool only.

## Acceptance Criteria

- [ ] `QFaultIRModule::dump(std::ostream& out) const` implemented
- [ ] For LOGICAL level: prints `; Module: <name> [LOGICAL]`, then each `LogicalGate` as `<GateKind> <qubit_name>(s)` (e.g., `H q[0]`, `CX q[0] q[,2]`, `T q[2]`)
- [ ] For PHYSICAL level: prints `; Module: <name> [PHYSICAL]`, then each `PatchOp` as `<OpKind> (<x>,<y>) ... @t=<timeStep>`
- [ ] Round-trip requirement: two modules constructed with identical instructions produce identical `dump()` output (tested in `test_QFaultIRModule.cpp`)
- [ ] `std::visit` used for instruction dispatch — no `if constexpr` chains
- [ ] `clang-tidy` clean

## Technical Details

- Output format is diagnostic only — it does not need to be parseable back
- No `std::cerr` — always writes to the provided `ostream`
- Add a convenience `module.dumpToString() const -> std::string` that returns `dump()` into a `std::ostringstream`

## Dependencies

- Task 007 (1-A-3): QFaultIRModule must exist
