# Task: 1-A-1 — LogicalQubit, GateKind, LogicalGate types

## Description

Define the logical-level IR types in `src/qfault/ir/`. These represent a quantum circuit at the abstract gate level (before any QEC mapping). See ADR-0001 for the design decisions.

## Acceptance Criteria

- [ ] `include/qfault/ir/LogicalQubit.hpp`: `struct LogicalQubit { std::string name; std::size_t index; };` with `operator==`
- [ ] `include/qfault/ir/GateKind.hpp`: `enum class GateKind { H, X, Y, Z, S, Sdg, T, Tdg, CX, CZ, RZ, U };`
- [ ] `include/qfault/ir/LogicalGate.hpp`: `struct LogicalGate { GateKind kind; std::vector<LogicalQubit> operands; std::optional<double> angle; };` with `operator==`
- [ ] `tests/unit/test_LogicalGate.cpp`: unit tests for construction, equality, and all GateKind values
- [ ] `clang-tidy` clean; `-Wall -Wextra -Wpedantic` clean on new files
- [ ] No raw `new`/`delete`; no `using namespace std;`
- [ ] `[[nodiscard]]` where applicable; `enum class` only (no plain enum)

## Technical Details

- Headers go in `include/qfault/ir/` (public API)
- Implementation files (if any) in `src/qfault/ir/`
- Use designated initialisers in tests: `LogicalGate{.kind=GateKind::T, .operands={q0}}`
- `std::optional<double> angle` is only set for rotation gates (RZ, U); others leave it nullopt
