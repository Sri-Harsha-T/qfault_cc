# Task: 1-C-2 — QASM 3.0 parser → QFaultIRModule

## Description

Implement the parser that converts a tokenised QASM 3.0 Clifford+T program into a `QFaultIRModule` at `IRLevel::LOGICAL`. This is a simple recursive descent parser — no AST intermediate representation needed.

## Acceptance Criteria

- [ ] `include/qfault/frontend/Parser.hpp` + `src/qfault/frontend/Parser.cpp` implemented
- [ ] `Parser::parse(std::string_view src) -> QFaultIRModule` — main entry point
- [ ] Handles: `OPENQASM 3.0;`, `qubit[N] name;`, `h name[i];`, `cx name[i], name[j];`, `t name[i];`, `tdg name[i];`, `s name[i];`, `sdg name[i];`
- [ ] Parsed qubits populate `QFaultIRModule::qubits` with correct names and indices
- [ ] Parsed gates populate `QFaultIRModule::instructions` as `LogicalGate` variants
- [ ] `QFaultIRModule::level` is set to `IRLevel::LOGICAL`
- [ ] Unknown/unsupported gate names: add `DiagLevel::Warn` diagnostic, skip the gate
- [ ] `tests/unit/test_Parser.cpp`: parse a 5-qubit, 10-gate Clifford+T program; verify qubit count, instruction count, and specific gate kinds
- [ ] `clang-tidy` clean; no exceptions in the parser hot path — use return-value errors

## Technical Details

- Parser owns a `PassContext`-style diagnostic list internally (or accept one as a parameter)
- Parse errors (syntax errors): add `DiagLevel::Error` and skip to next semicolon (error recovery)
- Do not use `std::cerr` — all diagnostics via the internal diagnostic list, returned with the module
