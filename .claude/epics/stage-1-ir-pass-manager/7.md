# Task: 1-A-2 — PatchCoord, MeasBasis, PatchOpKind, PatchOp types

## Description

Define the physical-level IR types in `src/qfault/ir/`. These represent surface code patch operations (the output of the LatticeSurgeryPass). See ADR-0001 and `.claude/rules/qec.md` for invariants.

## Acceptance Criteria

- [ ] `include/qfault/ir/PatchCoord.hpp`: `struct PatchCoord { int x, y; };` with `operator==` and `operator<` (for use in maps)
- [ ] `include/qfault/ir/MeasBasis.hpp`: `enum class MeasBasis { X, Z };` (Y is not a direct surface code measurement)
- [ ] `include/qfault/ir/PatchOpKind.hpp`: `enum class PatchOpKind { MERGE, SPLIT, MEASURE, IDLE };`
- [ ] `include/qfault/ir/PatchOp.hpp`: `struct PatchOp { PatchOpKind kind; std::vector<PatchCoord> patches; MeasBasis basis; std::size_t timeStep; };` with `operator==`
- [ ] `tests/unit/test_PatchOp.cpp`: unit tests for all types, construction, equality
- [ ] Runtime assertion in any code that uses code distance: `assert(d % 2 == 1 && d >= 3)`
- [ ] `clang-tidy` clean; no raw `new`/`delete`; `enum class` only

## Technical Details

- `PatchCoord` needs `operator<` for `std::map<PatchCoord, ...>` usage in later passes
- `MeasBasis::Y` is intentionally absent — Y measurements in surface code are composite operations
- `timeStep` is `std::size_t`, not `int` — negative time steps are a logic error

- Task 001 (CMakeLists.txt) — build system must exist
